# Frameworks and retention

A short funnel reference and practical churn ideas.

## Marketing funnel

Deep dive: [Semrush — marketing funnel](https://www.semrush.com/blog/marketing-funnel/)

**Compressed framework**

| Stage         | User state             | Typical metrics                              |
| ------------- | ---------------------- | -------------------------------------------- |
| Awareness     | Problem recognized     | Impressions, reach, branded search           |
| Interest      | Researching options    | Clicks, time on site, content downloads      |
| Consideration | Comparing vendors      | Trial starts, demo bookings, pricing visits  |
| Conversion    | Purchase or activation | Checkout completion, activation event        |
| Retention     | Repeat use             | DAU/WAU, retention curves, expansion revenue |
| Advocacy      | Referrals and reviews  | NPS, referral rate, UGC                      |

Pick one primary metric per stage; do not try to optimize everything at once. Define events and reports in [Measurement, attribution and experimentation](measurement-attribution-and-experimentation.md) so "awareness" and "conversion" mean the same thing to everyone.

## Churn reduction ideas

### Example (Apple TV pattern)

Forcing re-login and 2FA before sensitive actions (like subscription management) can cut accidental cancellations from kids or shared remotes — friction used on purpose, and only where it is fair to the user.

**SaaS analogues**

- Save flows on downgrade: pause, fewer seats, or a better-fitting plan.
- Exit surveys with one-click salvage offers (discount, training, export reassurance).
- Proactive success triggers when usage drops before renewal.
- Annual plans with clear value versus monthly hop-off.
